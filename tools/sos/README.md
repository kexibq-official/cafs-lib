# Sound of Sorting (CAFS edition)

Visualization that traces a CAFS-shaped sort through its four stages and renders an MP4 with sound and a silent GIF for the README.

## Output

- [`examples/cafs_demo.mp4`](examples/cafs_demo.mp4): video with audio. Each bucket update produces a short tone with frequency keyed to the rank of the value in the input palette. Open in any media player.
- [`examples/cafs_demo.gif`](examples/cafs_demo.gif): silent, palette-optimized GIF. This is the file embedded in the top README.

## Layout

The visualization splits the screen into three panels:

- **input** (top): the unsorted array as bars. The current element being read is highlighted in blue.
- **buckets** (middle): the hash table as `M x 4` cells. Each filled cell shows the running count for its key. The currently active slot gets a green halo on a hit and a purple halo on a miss.
- **output** (bottom): empty during sample and hot loop; painted left to right during emit.

Phase label at the top tracks the four stages: Chao1 sample, hot loop, reconstruct, emit.

## Reproduce

The driver compiles the tracer, generates events, renders frames + WAV, and runs ffmpeg for both MP4 and GIF outputs.

Windows PowerShell:

```
.\build.ps1
```

Linux / macOS / WSL:

```
./build.sh
```

Optional parameters (defaults shown):

```
.\build.ps1 -N 256 -K 8 -M 16 -Seed 7 -Fps 30
N=256 K=8 M=16 SEED=7 FPS=30 ./build.sh
```

Requires: g++ (any C++20-capable), Python with `numpy`, `scipy`, `matplotlib`, and `ffmpeg` on PATH.

## Files

| File           | Purpose                                                    |
|----------------|------------------------------------------------------------|
| `sos_trace.cpp`| CAFS-shaped sort that emits a JSON event log              |
| `render.py`    | Reads the event log, generates PNG frames and a WAV audio |
| `build.ps1`    | Windows driver                                             |
| `build.sh`     | Unix driver                                                |
| `events.json`  | Event log committed for the default run (regenerated by build) |

`sos_trace.cpp` is a simplified shape-of-CAFS: scalar bucket update, no SIMD specialization, no radix tail. It exists only to drive the visualization; the production sort is in [`include/cafs2.hpp`](../../include/cafs2.hpp).
